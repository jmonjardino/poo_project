# Semana 1 — Fundamentos

## Objetivo
- Montar ambiente, correr o projeto, criar 2–3 classes concretas neutras e validar registo/render/picking.

## Tarefas
- Alinhar `JDK` e `Gradle`; garantir build/run (`gradlew run`).
- Ler documentação do motor; entender que a camada de jogo é neutra (sem jME).
- Criar classes concretas simples (`Item`, `Terrain`, `Character` derivados) com construtores e getters.
- Registar pelo menos 1 objeto no `GameRegistry` e validar que aparece via `RenderAppState`.
- Organizar estrutura de pastas para `jogo.gameobject.*` e `jogo.voxel.*`.

## Entregáveis
- Código a compilar e correr; captura de ecrã da cena.
- UML inicial (classes e relações); Javadoc público básico.

## Critérios de aceitação
- Projeto executa com controlos base; objetos registados são visíveis.
- Classes de jogo não importam tipos do motor.
